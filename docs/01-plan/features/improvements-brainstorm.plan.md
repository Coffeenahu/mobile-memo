## 3. 우선순위 매트릭스

```
                높은 가치
                    |
        B1(정렬) A1(다크모드)
        C1(내보내기) C2(가져오기)
        D3(자동저장) E1(단축키)
        B5(복제) A2(색상라벨)
낮은 ───────────────────────── 높은 난이도
 난이도 |        B2(태그)  A5(스와이프)
        |   B3(체크리스트)
        C5(휴지통) D1(알림)
        B4(마크다운)
                    |
                낮은 가치
```

### 즉시 구현 가능 (쉽고 가치 높음)

1. ✅ **B1 — 정렬 옵션** (드롭다운 1개, 상태 1개)
2. ✅ **A1 — 다크모드** (CSS 변수 + prefers-color-scheme)
3. ✅ **C3 — 공유** (Web Share API)
4. ❌ **C1/C2 — 내보내기/가져오기** (구현불필요)
5. **D3 — 자동 저장** (debounce 300ms)
6. **E1 — 키보드 단축키** (keydown 이벤트 추가)

### 중기 구현 (가치 높고 난이도 중간)

6. ✅ **A2 — 색상 라벨** (카드 색상 바 + 편집 뷰 색상 선택)
7. ✅ **A5 — 스와이프** (touch 이벤트)
8. **B2 — 태그** (#태그 파싱 + 필터 칩)
9. **C5 — 휴지통** (삭제 플래그 + 별도 뷰)
10. **B3 — 체크리스트** (- [ ] 렌더링)

### 장기 검토 (큰 변경 또는 외부 의존)

10. **B4 — 마크다운 프리뷰** (파서 라이브러리 또는 직접 구현)
11. **D1 — 리마인더 알림** (Service Worker + Notifications API)
12. **C4 — 클라우드 동기화** (백엔드 필요, 레벨 업그레이드)

## 4. 다음 단계 제안

가장 빠르게 완성도를 높이는 순서:

```
✅ B1 (정렬) → ✅ A1 (다크모드) → ✅ C3 (공유) → ✅ A2 (색상라벨) → ✅ A5 (스와이프) → D3 (자동저장) → E1 (키보드 단축키)
```
